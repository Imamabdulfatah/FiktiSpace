## Securing Environment Variables

This project contains sensitive configuration values (API keys, database passwords, etc.) that **must never be committed to version control**. Follow these steps to keep them safe:

1. **Copy the example file**
   ```bash
   cp .env.example .env
   ```
   This creates a local `.env` file that will be ignored by Git (already listed in `.gitignore`).

2. **Generate a new application key**
   ```bash
   php artisan key:generate
   ```
   The command will populate `APP_KEY` in your `.env`.

3. **Fill in the remaining placeholders**
   Edit `.env` and provide real values for:
   - `DB_PASSWORD`
   - Any third‑party service tokens (e.g., AWS, Mailgun, Pusher)
   - `APP_SECRET` or other custom secrets your code may require

4. **Never commit `.env`**
   The `.gitignore` already contains `.env`, ensuring it stays out of the repository. If you ever need to share environment configuration with teammates, use a secure channel (e.g., encrypted password manager) and never paste secrets into pull requests.

5. **Deploy securely**
   When deploying, provide the same environment variables via your hosting platform's secret management (e.g., Laravel Forge, Docker secrets, CI/CD variables).

By following these steps you protect your application from accidental credential leakage.
